# Changelog — v3.0 (2026-03-27)

## Summary

Monorepo consolidation: removed legacy backends, promoted `rag_backend/` to `backend/`, and added UX-focused API improvements (structured errors, request tracing, rich health check).

## Changes

### Structure (Monorepo Refactor)

- **Deleted** `BackEnd/` (Flask v1) and `DialogFlow/` (FastAPI v1.5) — fully superseded
- **Deleted** stale root files: `load.py`, `resume*.md`
- **Renamed** `rag_backend/` → `backend/`
- **Removed** backward-compatibility shim modules (`app/extractors/`, `app/memory/`, `app/workflow/stages.py`, `app/workflow/response_generator.py`)
- **Restored** `app/workflow/stage_workflow.py` with direct `agent.*` imports (was real code, not a shim)
- **Moved** `.github/workflows/` to repo root; updated all CI path triggers

### UX / API Improvements

- **Health endpoint** (`GET /health`) now reports per-component status:
  - `agent`: ok / unavailable
  - `rag_index`: ok / unavailable (with error detail)
  - Overall: `ok` / `degraded` / `unhealthy`
- **Structured validation errors** (`422` responses): returns `{ errors: [{ field, message, type }] }` instead of raw Pydantic dump — easier for frontend to parse and display
- **Request-ID middleware**: every response includes `X-Request-ID` header for traceability; clients can pass their own via request header
- **Global error handler** now includes `request_id` in 500 responses for log correlation
- **API version** bumped to `2.0.0`; title updated to "AutoVend API"

### Tests

- **Added** `tests/test_storage.py` — 15 FileStorage unit tests (profile, session, test drive CRUD + edge cases)
- **Added** 4 new API tests: health component check, request-ID header/passthrough, structured validation error format
- **Updated** 3 test files (`test_base_extractor.py`, `test_extractors.py`, `test_workflow.py`) to import directly from `agent.*` instead of deleted shims

### Documentation

- **Created** root `README.md` with unified architecture overview
- **Updated** `backend/README.md` — removed `rag_backend` references, fixed CI link path
- **Updated** `.gitignore` — removed legacy entries, added coverage/cache patterns
- **Updated** `backend/pyproject.toml` — project name → `autovend-backend`

### Frontend

- **Updated** proxy target from `http://localhost:5000` → `http://localhost:8000`

## UX Optimization Rationale

| Improvement | Problem Solved | User Impact |
|---|---|---|
| Structured validation errors | Raw Pydantic errors are hard to parse on frontend | Frontend can display field-specific error messages |
| Request-ID header | No way to correlate frontend errors with backend logs | Support/debugging becomes traceable |
| Rich health check | `/health` only returned `{status: ok}` | Frontend/monitoring can detect degraded mode (e.g., RAG index missing) |
| API version in root | No version info visible | Clients can verify API compatibility |
